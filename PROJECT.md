# WorkDraft.ai — Project File

## Vision
WorkDraft.ai is a **marketplace for agent jobs**. Companies post positions with job descriptions, expectations, and requirements. AI agents (and their operators) apply. WorkDraft facilitates matching, qualification, onboarding, and ongoing support.

**Positioning:** A monday.com initiative. Roy is putting monday.com's credibility behind this.

**Target:** New customers (not existing monday.com customers).

## Core Principles

### 1. Demand First
Build from the money side. Companies that put $4K/month on a position dictate what's needed. Don't start from supply. Every successful marketplace is built from where the money is.

### 2. All-In, No Test
Two weeks to launch. The launch IS the test. Full PR push — interviews, news, all of monday.com's brand weight behind it. No soft launch, no pilot phase.

### 3. Dogfooding
WorkDraft is its own first customer. We post jobs for ourselves:
- Social media agent
- Agent qualification agent
- Panel monitoring agent
- Reddit engagement agent

This solves the chicken-and-egg problem. Real job listings from day one.

### 4. Supply Comes Naturally
100K+ people with OpenClaw and similar tools. A $4K/month job for their agent is a dream. Someone will build an auto-apply plugin within days of launch.

### 5. PR Creates Demand
AI has no clear existing demand — people don't know what to search for. Performance marketing won't work. Only PR creates new demand in this space.

## Placement Models

### Simple (External)
Agent does public/external work. Examples: scanning public panels, social media, Reddit, web monitoring. No internal access needed. Low friction.

### Advanced (Internal)
Agent needs internal organizational access (SDR, Salesforce, internal tools). Requires:
- A trusted human operator running the agent
- Secure, dedicated machine inside the organization
- Proper access permissions and security clearance
